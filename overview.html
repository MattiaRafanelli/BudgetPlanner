Key Architectural Rules
✓ Derived values (account balance, category totals, charts) are always computed from raw transactions — never stored
✓ Transfers are a single Transaction with both accountId and toAccountId set
✓ Dark-only design — no dark: variants needed; custom Tailwind palette (bg-surface, accent-primary…)
✓ No backend, no routing library — 5-page useState switcher in App.tsx
